一、项目结构总览
本项目采用前后端分离架构，后端提供 RESTful JSON 接口，前端 Vue3 SPA 消费接口。整体分为「前台博客展示」和「后台管理」两个端，共用同一套后端服务。

二、数据库设计（MySQL 8.0）
共设计 6 张核心表，使用逻辑删除（is_deleted 字段），所有时间字段使用 datetime 类型。

2.1 ER 关系说明
文章（article）属于一个分类（category），拥有多个标签（article_tag 中间表）
文章下可有多条评论（comment），评论支持二级嵌套（parent_id 自关联）
用户（user）表仅存管理员账号，前台无需注册

4.2 Axios 封装（src/api/request.ts）
// src/api/request.ts
import axios from &apos;axios&apos;
import { useUserStore } from &apos;@/stores/user&apos;

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,  // .env 注入
  timeout: 10000,
})

// 请求拦截器：自动附加 Token
request.interceptors.request.use(config =&gt; {
  const { token } = useUserStore()
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截器：统一错误处理
request.interceptors.response.use(
  res =&gt; res.data,           // 直接返回 data 层
  err =&gt; {
    if (err.response?.status === 401) {
      useUserStore().logout()
      router.push(&apos;/admin/login&apos;)
    }
    return Promise.reject(err)
  }
)

五、核心页面实现要点
5.1 文章编辑页（ArticleEdit.vue）
这是整个项目技术含量最高的页面，重点实现以下功能：
使用 md-editor-v3 或 @bytemd/vue-next 作为 Markdown 编辑器，支持实时预览
封面图上传：调用 /admin/upload 接口，上传后回填 URL
标签选择：Element Plus 的 el-select 多选模式，数据从 Pinia 缓存
自动保存草稿：使用 watch + debounce(3000ms)，静默调用 updateArticle
发布/保存 两个按钮分别传 status=1 和 status=0

// composables/useArticleEditor.ts
export function useArticleEditor(articleId?: number) {
  const form = reactive&lt;Partial&lt;Article&gt;&gt;({ status: 0, tagIds: [], allowComment: true })
  const loading = ref(false)

  // 自动保存草稿（防抖）
  const autoSave = useDebounceFn(async () =&gt; {
    if (!form.title || !form.id) return
    await updateArticle(form.id, { ...form, status: 0 })
  }, 3000)

  watch(() =&gt; form.content, autoSave)
  return { form, loading, autoSave }
}

            .allowedHeaders(&quot;*&quot;)
            .allowCredentials(true);
    }
}

八、服务器部署方案
8.1 后端部署
打包：mvn clean package -DskipTests 生成 blog.jar
上传到服务器：/opt/blog/blog.jar
启动：nohup java -jar /opt/blog/blog.jar --spring.profiles.active=prod &gt; /opt/blog/app.log 2&gt;&amp;1 &amp;
systemd 服务文件：配置开机自启，监控自动重启

8.2 前端部署
打包：npm run build，生成 dist/ 目录
上传 dist/ 到服务器：/var/www/blog/
Nginx 配置：前台 / 指向 dist/index.html，/api/ 反代到 Spring Boot :8080

# Nginx 配置示例
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/blog;
    index index.html;

    # 前端 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 上传文件静态访问
    location /uploads/ {
        alias /var/blog/uploads/;
    }
}
