# 个人博客 CMS 需求说明书

## 一、项目概述

个人博客 CMS 系统用于个人创作者发布文章、管理内容、维护用户互动和配置站点信息。系统分为前台展示端和后台管理端两部分，前台面向访问用户，后台面向管理员。

## 二、用户角色

| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| 游客 | 未登录访问者 | 浏览文章、搜索、查看分类标签、查看友链、提交留言 |
| 前台用户 | 注册登录用户 | 评论、点赞、收藏、查看用户中心、修改个人资料 |
| 管理员 | 后台维护人员 | 管理文章、分类、标签、评论、用户、媒体、站点设置等 |

## 三、功能需求

### 1. 前台展示需求

1. 首页展示文章列表、分类、标签和每日一句。
2. 支持按关键词搜索文章。
3. 支持根据分类或标签筛选文章。
4. 支持查看文章详情，包括标题、摘要、正文、浏览量、点赞数、评论数等。
5. 支持文章归档，方便按时间浏览历史内容。
6. 支持专题页，将分类和标签进行聚合展示。
7. 支持友情链接展示和申请。
8. 支持留言板功能。
9. 支持用户注册、登录、个人中心、修改资料和修改密码。
10. 支持文章收藏和点赞。

### 2. 后台管理需求

1. 管理员登录后进入后台管理系统。
2. 仪表盘展示文章、分类、标签、评论等统计数据。
3. 支持文章新增、编辑、删除、预览和封面上传。
4. 支持分类新增、编辑、删除、排序和文章数量统计。
5. 支持标签新增、编辑、删除、颜色设置和排序。
6. 支持评论审核、拒绝、回复和删除。
7. 支持留言审核、回复和删除。
8. 支持前台用户启用、禁用和重置密码。
9. 支持媒体库图片上传、分类、复制地址和删除。
10. 支持友情链接审核、新增、编辑、删除和排序。
11. 支持站点名称、Logo、头像、首页文案、每日一句和路线图配置。
12. 支持操作日志查询，记录管理员关键操作。
13. 支持 Skills 管理，用于维护文章编辑相关扩展能力。

## 四、非功能需求

### 1. 技术需求

- 前端必须使用 Vue 3。
- 使用 TypeScript 编写关键页面、接口和数据类型。
- 使用 Vite 作为构建工具。
- 使用 Axios 完成网络请求。
- 前后端交互数据格式为 JSON。

### 2. 质量需求

- 命名采用驼峰命名法。
- 页面逻辑、接口封装和复杂计算需要添加注释。
- API 调用统一封装，降低页面和接口的耦合。
- 页面需要具备加载状态和错误提示。
- 后台重要页面需要登录权限控制。

### 3. 安全与内容要求

- 项目内容积极健康，不包含违法违规内容。
- 后台管理页面需要管理员 Token 才能访问。
- 用户登录信息保存在本地，并通过请求拦截器自动携带。

## 五、数据需求

主要数据对象包括：

- 文章 Article
- 分类 Category
- 标签 Tag
- 评论 Comment
- 前台用户 FrontUser
- 管理员用户 AdminUser
- 媒体资源 MediaAsset
- 友情链接 FriendLink
- 留言 Message
- 站点设置 SiteSetting
- 操作日志 AuditLog
- Skills 配置 AiSkill

## 六、验收标准

1. 前台页面可以正常浏览文章、搜索、筛选和查看详情。
2. 后台页面可以正常登录并进行内容管理。
3. 至少包含 3 个以上新增功能模块。
4. 可以通过 Axios 请求后端接口并接收 JSON 数据。
5. 项目可以通过 `npm run build` 构建。
6. 答辩时可以演示主要功能和讲解核心代码。
