# 个人博客 CMS 系统设计说明书

## 一、总体架构

系统采用前后端分离思想。前端使用 Vue 3 + TypeScript + Vite 开发，负责页面展示、交互逻辑、路由控制和接口调用。后端提供 REST 风格接口，前端通过 Axios 请求后端服务，数据格式采用 JSON。

```text
用户浏览器
  |
  | Vue 3 + Vue Router + Pinia
  |
  | Axios / JSON
  |
后端 API 服务
  |
数据库 / 文件上传目录
```

## 二、前端目录结构

```text
blog-frontend/src
├─ api          接口封装
├─ components   公共组件
├─ composables  组合式函数
├─ layouts      后台布局
├─ pages        页面组件
│  ├─ admin     后台管理页面
│  └─ front     前台展示页面
├─ router       路由配置
├─ stores       Pinia 状态管理
├─ types        TypeScript 类型定义
└─ style.css    全局样式
```

## 三、路由设计

## 四、接口设计

前端所有请求通过 `src/api/http.ts` 中的 Axios 实例发送。该文件主要负责：

1. 设置统一请求前缀 `/api`。
2. 请求发出前自动读取本地 Token。
3. 区分前台用户 Token 和后台管理员 Token。
4. 响应失败时统一处理 401、超时和后端错误消息。

示例：

```ts
const http = axios.create({
  baseURL: "/api",
  timeout: 10000
});
```

## 五、类型设计

项目在 `src/types` 中定义了主要业务类型，例如：

- `Article`：文章数据结构
- `Category`：分类数据结构
- `Tag`：标签数据结构
- `CommentItem`：评论数据结构
- `SiteSetting`：站点设置数据结构
- `AdminUser`：后台用户管理数据结构
- `AiSkill`：Skills 配置数据结构

TypeScript 类型可以帮助开发阶段提前发现字段错误，提高代码可靠性。

## 六、核心组件和页面设计

### 1. AdminLayout.vue

后台管理系统统一布局组件，负责侧边栏导航、顶部栏、当前页面标题、管理员退出登录和修改密码弹窗。

### 2. Dashboard.vue

后台仪表盘页面，负责把文章、评论、分类和标签数据加工成统计卡片、饼图、折线图和柱状图。

### 3. ArticleEdit.vue

文章编辑页面，支持文章标题、摘要、正文、封面、分类、标签、SEO 信息和 Markdown 编辑。

### 4. About.vue

前台关于页，展示站点信息、文章统计、互动数据、路线图和最近文章。

## 七、状态管理设计

项目使用 Pinia 管理部分全局状态：

- 管理员登录状态
- 前台用户登录状态
- 后台侧边栏折叠状态

普通页面内部状态使用 Vue 3 的 `ref`、`reactive` 和 `computed` 管理。

## 八、权限设计

路由守卫会检查本地是否存在管理员 Token：

- 访问 `/admin` 开头的后台页面时，如果没有管理员 Token，则跳转到 `/login`。
- 已登录管理员访问 `/login` 时，自动跳转到 `/admin/dashboard`。
- 前台用户中心需要普通用户 Token，没有登录时跳转到 `/user/login`。

## 九、设计总结

本系统结构清晰，前台和后台职责分离，接口封装统一，数据类型明确，适合作为 Vue 3 课程结课项目进行展示和答辩。
