# 个人博客 CMS 答辩讲稿

## 一、开场介绍

老师好，我的结课项目是“个人博客 CMS 系统”。这个项目基于 Vue 3 和 TypeScript 开发，使用 Vite 作为构建工具，使用 Axios 完成前后端网络请求，项目整体分为前台展示端和后台管理端两部分。

本项目主要面向个人创作者，用来发布技术文章、管理分类标签、维护评论留言、配置站点信息和查看后台统计数据。

## 二、项目背景

我选择个人博客 CMS 作为结课项目，是因为它既包含前台页面展示，也包含后台数据管理，能比较完整地体现移动互联网应用开发中常见的路由、组件、请求、状态管理和表单交互。

项目内容以技术文章和个人学习记录为主，内容积极健康，符合课程要求。

## 三、技术栈介绍

本项目主要使用以下技术：

1. Vue 3：用于构建前端页面和组件。
2. TypeScript：用于定义接口类型和业务数据类型，提高类型安全。
3. Vite：用于项目开发和构建。
4. Vue Router：用于前台和后台页面路由管理。
5. Pinia：用于保存用户登录状态和后台布局状态。
6. Axios：用于发送网络请求，完成前后端 JSON 数据交互。

## 四、功能模块介绍

项目分为前台和后台。

前台主要包括：首页、文章详情、搜索、归档、专题、友链、留言板、关于页、用户登录注册、用户中心、收藏和点赞等功能。

后台主要包括：管理员登录、仪表盘、文章管理、分类管理、标签管理、评论管理、留言管理、用户管理、媒体库、站点设置、操作日志和 Skills 管理等功能。

## 五、项目演示流程

### 1. 前台演示

首先打开首页，展示文章列表和站点信息。然后进入文章详情页，演示文章阅读、点赞、收藏和评论功能。接着展示搜索页面、分类标签筛选、归档页、专题页和关于页。最后展示用户登录注册和用户中心。

### 2. 后台演示

进入后台登录页，使用管理员账号登录。登录后首先展示仪表盘，可以看到文章统计、评论统计和发布趋势图。然后演示文章管理，新增或编辑一篇文章。接着演示分类、标签、评论、留言、用户、媒体库和站点设置。最后展示操作日志，说明管理员关键操作会被记录。

## 六、核心代码讲解

我重点讲解 Axios 请求封装和路由权限控制。

在 `src/api/http.ts` 中，我创建了统一的 Axios 实例：

```ts
const http = axios.create({
  baseURL: "/api",
  timeout: 10000
});
```

请求拦截器会在请求发出前自动读取本地 Token，并放入 Authorization 请求头。系统区分前台用户 Token 和后台管理员 Token，因此前台用户接口和后台管理接口可以分别维护登录状态。

响应拦截器负责统一处理错误，例如 401 登录失效、请求超时和后端错误消息。这样每个页面就不需要重复写大量错误处理代码。

路由权限控制在 `src/router/index.ts` 中完成。当用户访问 `/admin` 开头的后台页面时，如果本地没有管理员 Token，就跳转到 `/login`。这样可以防止未登录用户直接进入后台管理页面。
