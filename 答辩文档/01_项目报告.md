# 《移动互联网应用技术》结课项目报告

## 一、项目基本信息

- 项目名称：个人博客 CMS 系统
- 项目类型：前后台一体化内容管理系统
- 前端技术栈：Vue 3、TypeScript、Vite、Vue Router、Pinia、Axios
- 后端交互方式：Axios 发送 HTTP 请求，使用 JSON 数据格式与后端服务交互
- 项目定位：面向个人创作者的博客发布、内容管理、互动评论和站点配置平台

## 二、项目背景

随着移动互联网内容平台的发展，个人创作者需要一个可以独立维护内容、展示作品、管理评论和沉淀用户互动的轻量级系统。本项目以个人博客为主题，设计并实现了前台展示页面和后台 CMS 管理页面，满足文章发布、分类标签、评论留言、用户中心、媒体管理、站点设置等常见业务需求。

项目内容积极健康，主要用于技术文章、学习笔记、项目记录和个人成长内容展示，符合课程项目要求。

## 三、项目目标

1. 使用 Vue 3 和 TypeScript 完成一个结构清晰、功能完整的前端项目。
2. 使用 Vite 作为构建工具，提高开发和打包效率。
3. 使用 Axios 完成前后端数据交互，统一处理请求、Token 和异常提示。
4. 将前台展示和后台管理分离，提升系统可维护性。
5. 通过组件复用、类型定义和代码注释提升代码质量。

## 四、系统功能概述

### 1. 前台功能

- 首页文章流展示
- 文章详情阅读
- 搜索文章
- 分类文章筛选
- 标签文章筛选
- 归档页
- 专题页
- 友链展示和申请
- 留言板
- 用户登录、注册和用户中心
- 收藏、点赞和评论互动
- 关于页展示站点信息和项目路线图

### 2. 后台功能

- 管理员登录
- 仪表盘数据统计
- 文章新增、编辑、删除、预览
- 分类管理
- 标签管理
- 评论审核和回复
- 留言审核和回复
- 用户管理
- 媒体库管理
- Skills 管理
- 友链审核和管理
- 站点设置
- 操作日志查看

## 五、技术实现说明

项目采用 Vue 3 的单文件组件结构，使用 `<script setup lang="ts">` 编写页面逻辑。路由由 `vue-router` 管理，后台页面统一挂载在 `/admin` 路由下，并通过路由守卫判断管理员登录状态。

网络请求统一封装在 `src/api/http.ts` 中，Axios 实例设置了 `baseURL: "/api"`，并通过请求拦截器自动携带 Token，通过响应拦截器统一处理后端返回结果和异常信息。

项目中的接口返回数据均按照 JSON 格式处理，前端通过 TypeScript 类型文件定义文章、分类、标签、评论、用户、媒体、站点设置等数据结构，提高代码可读性和类型安全。

## 六、项目特色

1. 前后台功能完整，既能展示博客内容，也能进行后台维护。
2. 使用 TypeScript 定义关键接口和数据模型，符合课程加分方向。
3. Axios 请求封装统一，减少页面重复代码。
4. 后台仪表盘包含文章状态、评论状态、发布趋势、分类标签统计等图表化展示。
5. 支持站点设置，可在后台维护前台名称、头像、简介和路线图。
6. 支持 Skills 管理，为文章编辑提供扩展能力。

## 七、总结

本项目围绕“个人博客 CMS”主题，综合使用 Vue 3、TypeScript、Vite、Vue Router、Pinia 和 Axios，完成了一个可演示、可维护、功能较完整的移动互联网应用课程项目。通过本项目开发，我熟悉了 Vue 3 项目结构、组件开发、路由管理、状态管理、网络请求封装和前后端数据交互流程。
